# 1. Purpose

This document defines the reusable UI components that shall be used throughout Gym ERP. All screens shall use these standardized components to maintain a consistent, modern, and professional interface.

---

# 2. Buttons

The application shall support the following button types:

* Primary
* Secondary
* Success
* Warning
* Danger
* Ghost
* Outline
* Icon Button

Every button shall support:

* Hover state
* Active state
* Disabled state
* Loading state

---

# 3. Input Components

The application shall provide:

* Text Input
* Password Input
* Number Input
* Email Input
* Phone Number Input
* Search Input
* Text Area

Every input shall support:

* Labels
* Placeholder text
* Validation
* Error messages
* Disabled state

---

# 4. Selection Components

The application shall include:

* Dropdown
* Multi-Select Dropdown
* Checkbox
* Radio Button
* Toggle Switch

---

# 5. Data Display Components

The application shall include:

* Data Tables
* Cards
* Statistic Cards
* Badges
* Avatars
* Progress Bars
* Tooltips

---

# 6. Navigation Components

The application shall include:

* Sidebar
* Top Navigation Bar
* Breadcrumbs
* Tabs
* Pagination
* Menu Items

---

# 7. Feedback Components

The application shall include:

* Toast Notifications
* Alert Messages
* Confirmation Dialogs
* Success Messages
* Error Messages
* Loading Indicators

---

# 8. Modal Components

The application shall support:

* Confirmation Modals
* Information Modals
* Form Modals
* Delete Confirmation Dialogs

---

# 9. Dashboard Components

The dashboard shall use:

* Statistics Cards
* Charts
* Activity Timeline
* Quick Action Cards
* Recent Activity Table

---

# 10. Standard Behavior

All components shall:

* Follow the official color palette.
* Follow typography standards.
* Support Light and Dark Mode.
* Maintain consistent spacing.
* Display smooth animations.
* Provide visual feedback for user interaction.

---

# 11. Reusability

Every component shall be designed once and reused throughout the application to ensure consistency and reduce maintenance effort.

---

# 12. Conclusion

The Component Library serves as the foundation of the Gym ERP user interface and shall be used across all modules to deliver a consistent and premium user experience.
